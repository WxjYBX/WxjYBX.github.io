1. Selenium架构(概述 | Selenium)

1.1. WebDriver(APIs)

概述

  1. 用于桌面网站测试自动化
  2. 使用浏览器供应商提供的浏览器自动化 API 来控制浏览器和运行测试
  3. WebDriver是通过浏览器本身提供的API来操纵浏览器，可以与不同的浏览器进行交互而无需修改应用程序的代码

注意

  1. WebDriver 有且只有一个任务: 与浏览器通信。 测试与它无关。
  2. 需要一个与绑定语言相匹配的测试框架，eg. NET 的 NUnit, Java 的 JUnit, Ruby 的 RSpec …

1.2. Selenium IDE

概述

  1. Web Ready：提供简单的、即插即用的解决方案，能够快速编写出在任何浏览器上都能够使用的端到端的测试用例
  2. Easy Debugging：IDE支持断点调试与中断功能
  3. Cross-browser Execution：支持跨浏览器运行，浏览器运行组件的时候可以使用IDE的命令行进行控制

1.3. Grid

概述

  1. 跨平台、跨设备：允许您在不同平台的不同机器上运行测试用例
  2. 远程使用：可以本地控制测试用例的操作, 当测试用例被触发时, 它们由远端自动执行

2. Selenium的通信原理

2.1. 直接通信

概念

WebDriver 通过驱动程序向浏览器传递命令， 然后通过相同的路径接收信息

WebDriver：Selenium

Driver：浏览器的驱动程序，chrome/firefox/..都有，eg. ChromeDriver –谷歌的 Chrome/Chromium， GeckoDriver – Mozilla 的 Firefox

Browser：浏览器，使用浏览器驱动程序构建的浏览器，用于自动访问

2.2. 远程通信

概念

WebDriver 通过 RemoteWebDriver/Selenium 服务器/Selenium Grid 对驱动程序操作，向浏览器传递命令， 然后通过相同的路径接收信息

3. Selenium的功能

3.1. 核心

  1. web 浏览器自动化的一个工具集， 远程控制浏览器实例， 模拟用户与浏览器的交互
  2. 浏览器用户代理库
  3. eg. 将文本输入到字段中，选择下拉值和复选框，并单击文档中的链接，鼠标移动、任意 JavaScript 执行

3.2. 特点

支持所有（主要）浏览器技术的通用接口

4. Selenium初体验

4.1. 前置准备

  1. 相关库(安装Selenium类库 | Selenium)–支持Java(Maven、Kotlin)、Python(PyPI · The Python Package Index)、JS、C#、Ruby

  2. 浏览器与相关驱动

    1. chrome
      1. 驱动下载1：chromedriver.storage.googleapis.com/index.html
      2. 驱动下载2：ChromeDriver - WebDriver for Chrome - Downloads

    2. firefox
      1. 驱动下载：Releases · mozilla/geckodriver (github.com)
      2. 版本驱动对应关系：Supported platforms — Firefox Source Docs documentation (mozilla.org)

  6. 操作元素
